Class Method Details

.call(collection, request, **options) ⇒ Hash

使用 Rails.cache.fetch，先嘗試尋找 cache 並回傳 cached 的內容，若沒有才透過 ActiveModelSerializers::SerializableResource#as_json 來產生 API render 時需要的資料。

Examples:

要一併 render 出 Article 的 Comments，並在 meta 中加入 comment_count

# API::V1::ArticlesController#index
# ...
  result = Api::CheckCachedDataService.call(@articles,
                                            request,
                                            meta: { article_count: Article.count },
                                            extra: { include: [:comments],
                                                     show_comments: true })
  render json: result
# ...

Parameters:

  • collection (Object) —

    ActiveRecord::Relation 物件或是單筆 model 的 instance。通常是使用 FetchingDataService 回傳的內容

  • request (Object) —

    從 controller 中得到的 request 物件

  • options (Hash) —

    可傳入要給 AMS 的額外 options。 options[:meta] 會加入 AMS 的 meta 中，而 options[:extra] 則是 AMS 的 include 內容與 instance_options

Returns:

  • (Hash) —

    會回傳 Cache 或直接透過 ActiveModelSerializers::SerializableResource#as_json 得到的內容

See Also:

  • FetchingDataService.call

def self.pagination_dict(collection, request)
  return {} if request.params[:page].blank?

  {
    current_page: collection.current_page,
    next_page: collection.next_page,
    prev_page: collection.prev_page, # use collection.previous_page when using will_paginate
    total_pages: collection.total_pages,
    total_count: collection.total_count
  }
end

def call
  Rails.cache.fetch(collection.cache_key) do
    show_indicator_in_development_log
    ActiveModelSerializers::SerializableResource.new(collection, structured_options).as_json
  end
end

def structured_options
  @options ||= {
    meta: pagination_dict.merge(meta_options),
    serialization_context: ActiveModelSerializers::SerializationContext.new(request)
  }.merge(extra_options)
end
